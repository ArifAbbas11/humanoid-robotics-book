# Contributing to Physical AI & Humanoid Robotics Book

Thank you for your interest in contributing to the Physical AI & Humanoid Robotics Book! This document outlines the process for contributing to this open-source book project.

## How to Contribute

### Reporting Issues
- Report errors, typos, or unclear explanations as GitHub issues
- Provide specific details about the problem and suggest improvements
- Include the location (file and line number) where the issue occurs

### Contributing Content
1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-content`)
3. Make your changes
4. Add or update tests if applicable
5. Commit your changes (`git commit -m 'Add amazing content'`)
6. Push to the branch (`git push origin feature/amazing-content`)
7. Open a Pull Request

### Content Guidelines
- Write for beginner-to-intermediate developers
- Include clear examples and code snippets
- Ensure all technical instructions are reproducible
- Use consistent formatting and style
- Add proper attribution for diagrams and figures

### Technical Requirements
- All code examples must be tested and functional
- Instructions must work on Ubuntu 22.04, Windows, and macOS
- Follow the existing documentation structure
- Maintain the 12,000-18,000 word count range

## Style Guide

### Writing Style
- Use clear, concise language
- Explain technical concepts with analogies when helpful
- Include learning objectives at the beginning of each section
- Provide summaries and practice tasks at the end of each section

### Code Examples
- Include complete, copy-paste ready code samples
- Add comments to explain complex concepts
- Test all examples in clean environments
- Follow consistent formatting and naming conventions

## Development Setup

1. Clone the repository
2. Install Node.js 18+
3. Run `npm install` to install dependencies
4. Run `npm start` to start the development server

## Pull Request Process

1. Update the documentation with details of changes
2. Ensure all code examples are functional
3. Add yourself to the contributors list if desired
4. Wait for review and address feedback

## Questions?

If you have questions about contributing, feel free to open an issue for discussion.
